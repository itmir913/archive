HTML 강의노트를 시작한 이유 — PPT에서 웹으로

July 12, 2026 · CS Education

PPT 작성 피로와 컴퓨터실 환경이 만나 GitHub Pages 기반 웹 강의노트 시스템이 됐다

July 12, 2026 CS Education cs-education

배경: PPT의 한계

매 수업마다 PPT를 새로 만드는 일은 생각보다 많은 시간을 소모한다. 디자인을 맞추고, 슬라이드를 구성하고, 내용이 바뀔 때마다 다시 손봐야 한다. 수업 내용 자체를 준비하는 것과 별개로 PPT를 만드는 노고가 따로 존재했다.

컴퓨터실이라는 조건

정보 교과 수업은 컴퓨터실에서 이루어진다. 학생들 앞에 PC가 있다. 이 환경에서 종이 학습지나 교사 화면의 PPT를 보게 하는 것보다, 학생들이 자기 화면에서 직접 강의노트를 읽고 실습하는 방식이 더 자연스럽다는 판단을 했다.

결정: 2026년 3월 초

GitHub Pages를 이용해 웹에 강의노트를 게시하기로 했다. HTML + Tailwind CSS로 작성하면 버전 관리가 되고, 브라우저에서 바로 열리고, 별도 배포 인프라도 필요 없다.

처음엔 단순하게 시작했다. 기존 수업 내용을 HTML로 옮기는 것이 전부였다.

예상보다 길어진 여정

강의노트를 만들다 보니 인공지능기초 수업에서 시뮬레이터가 필요해졌다. 탐색 알고리즘, 지도학습, 강화학습 같은 개념을 설명만으로 전달하는 것에는 한계가 있었다. 직접 값을 바꾸고 결과를 보는 인터랙티브 시뮬레이터가 있어야 했다.

한국어 시뮬레이터 자료가 없다는 것을 확인하고, 직접 만들기로 했다. 간단하게 시작된 강의노트가 시뮬레이터 제작까지 이어졌다.

수행평가 docx 생성기도 같은 흐름에서 나왔다. 강의노트에서 수행평가 양식 다운로드 버튼을 연결하다 보니, 양식 자체를 코드로 생성하는 시스템이 필요해졌다.

현재

인공지능기초(2015개정) 3단원 강의노트 6개 파일, 프로그래밍(Python) 1·2단원 완료, AI 시뮬레이터 16개, 수행평가 docx 생성기로 이어졌다. 3월 초에 내린 결정 하나가 지금의 _Visual_Assets 레포지토리 전체의 출발점이다.
