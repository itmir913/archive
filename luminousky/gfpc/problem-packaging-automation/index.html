문제 패키징 자동화 — 엑셀과 Python으로 DOMjudge 진입장벽 낮추기

July 17, 2026 · GFPC Programming Challenge

SQL을 직접 만지지 않고도 문제와 계정을 패키징할 수 있는 도구를 만든 이유

July 17, 2026 GFPC Programming Challenge gfpc

맥락

DOMjudge에 문제를 등록하려면 원래 테스트케이스 파일을 사람이 하나하나 만들어서 패키징해야 한다. 계정을 여러 개 만드는 것도 마찬가지로 수작업이 기본이다.

이 부분에 참고할 만한 자료를 찾아봤는데, DOMjudge 공식 사이트에도 일괄 생성 관련 도구는 마땅히 없었다. 정보 교과 카페에서 과거 선배 교사들의 사례를 봐도, 데이터베이스에 SQL을 직접 입력해서 계정을 만드는 방식이었다. 이건 리눅스나 SQL 지식이 없는 사람은 따라 하기 어려운 방법이다.

고려한 대안

딱히 다른 대안은 없었다. 기존에 존재하는 참고 자료 자체가 거의 없었다는 것이 오히려 이 작업을 시작한 이유였다.

결정

Claude AI와 함께 엑셀 시트 + Python 스크립트 조합으로 자동화 도구를 만들었다.

  • 문제 출제자가 엑셀 템플릿에 문제 정보와 테스트케이스를 입력하면, Python 스크립트가 DOMjudge가 요구하는 패키징 형식으로 변환한다.
  • 학생 계정도 일괄로 생성할 수 있는 별도 Python 스크립트를 만들었다.

결과

리눅스나 SQL 지식이 없는 사람이라도 엑셀 시트를 채우고 스크립트를 실행하는 것만으로 DOMjudge에 문제를 올리고 계정을 만들 수 있는 기반을 마련했다. 참고할 만한 자료 자체가 없던 상황에서, 앞으로 이어받을 사람이 좀 더 쉽게 시작할 수 있는 방법 하나를 만들어둔 셈이다.

여름방학에는 이 도구를 한 단계 더 발전시켜, ICPC 형태의 DOMjudge 문제를 만들기 위한 프로젝트를 새로 시작할 계획이다. 계정 일괄 생성과 문제 패키징을 더 도와주는 GUI 형태의 프로그램을 구상하고 있다. 목표는 내년에 다른 사람이 GFPC를 이어받거나, 다른 지역에서 같은 방식을 시도할 때 조금 더 쉽게 시작할 수 있도록 초석을 놓는 것이다.
