## 배경

학생들에게 수행평가 예시 샘플 문서를 제공하려면 내가 직접 디자인하고 내용을 채워야 한다. 프로젝트마다 양식이 조금씩 다르고, 수정이 생기면 파일을 하나씩 열어서 고쳐야 한다.

처음엔 **메일머지** 기능을 검토했다. 템플릿 하나에 데이터를 끼워 넣는 방식이라 관리는 쉬울 것 같았다. 그런데 양식 구조 자체가 복잡하고, Word 메일머지로는 표·단락 구조를 세밀하게 다루기 어렵다는 판단이 들었다.

아예 DOCX 파일 자체를 코드로 생성하는 방향으로 바꿨다. `docx` npm 패키지를 쓰면 문서 구조를 코드로 정의할 수 있고, 프로젝트별 데이터만 교체하면 동일한 양식으로 여러 파일을 만들 수 있다. AI의 도움을 받아 템플릿을 짜고 내용을 채우면 혼자 직접 디자인하는 것보다 빠르게 만들 수 있다는 기대감도 있었다.

이 판단 — "학습지를 코드로 만든다" — 이 이후 `teacher-workshops` 프로젝트의 원천이 됐다. 수업 학습지를 JS로 생성하는 방식은 같은 맥락에서 나왔다.

## 개요

`_Visual_Assets` 레포지토리는 정보 교과 수업에서 쓰는 교수학습 자료를 통합 관리한다. 세 가지 유형으로 나뉜다:

1. **HTML 강의노트** — 수업 중 빔프로젝터로 띄우는 인터랙티브 웹 강의안
2. **수행평가 docx 자동생성기** — 프로젝트 양식 Word 문서 생성 Node.js 스크립트
3. **AI 기계학습 보고서 예시** — 학생용 참고 보고서 docx 자동생성기

## HTML 강의노트

### 공통 디자인 시스템

모든 강의노트는 하나의 캐노니컬 디자인 스택을 공유한다:

- **CSS**: Tailwind CSS CDN (바닐라 CSS 제거)
- **아이콘**: Lucide (Font Awesome 제거)
- **코드 강조**: Prism.js 1.29.0 (수동 `<span>` 태그 제거)
- **폰트**: Pretendard Variable (Google Fonts 제거)
- **최소 글씨 크기**: `text-base` (1rem)

헤더 그라디언트 (`bg-gradient-to-r from-blue-600 via-indigo-600 to-indigo-700`), section-card, quiz-btn 등 공통 컴포넌트를 모든 파일에 통일 적용했다.

### 교과별 현황

**인공지능기초(2015개정)**
- 3단원(데이터와 기계학습) 강의노트 완성: `3-1-1`~`3-2-2` (6개 파일)
- 주요 내용: 데이터 속성, 시각화, 정형/비정형, 전처리, 분류 모델, 기계학습 모델 구현

**프로그래밍(Python)**
- 1단원·2단원 전 파일 캐노니컬 디자인 통일 완료 (약 12개 파일)
- 3단원은 구형 바닐라 CSS 스택 → Tailwind 전환 대상

**프로그래밍(C)**
- 캐노니컬 기준 파일 (`2-4.제어문-1-선택구조.html`) 존재

## 수행평가 docx 자동생성기

Python 프로그래밍 프로젝트 수행평가 양식을 프로젝트별로 자동 생성한다.

### 파일 구조

```
templates/
  make/
    make_sw_template.js       ← 공통 라이브러리 (makeDocument() export)
    make_3-2-N_py.js          ← Python 프로젝트별 데이터
    make_3-2-N_c.js           ← C 프로젝트별 데이터
    make_ai_template.js       ← AI 보고서 공통 라이브러리
    make_ai_N.js              ← AI 보고서 주제별 데이터
  py/                         ← 생성된 Python 수행평가 docx
  c/                          ← 생성된 C 수행평가 docx
  ai/                         ← 생성된 AI 보고서 docx
```

### 수행평가 섹션 구조 (s1~s5)

| 섹션 | 내용 |
|------|------|
| s1 기획서 | 프로그램명, 목적, 대상 사용자, 주요 기능, 화면 예시 |
| s2 설계서-1 | 입출력 설계, 입출력 예시, 제약 조건 |
| s3 설계서-2 | 순서도, 의사코드 |
| s4 구현 | 소스코드, 코드 설명 |
| s5 테스트 | 테스트 케이스, 발견 오류, 개선점 |

- 라이브러리: `docx` npm 패키지
- DXA 단위 기준: A4(`11906×16838`), 여백 1000, 컨텐츠 폭 9906

### 수행평가 프로젝트 목록 (Python 3단원, 15개)

비만도 측정, 일차방정식, 이달은 며칠까지, 철근 자르기, 바코드 검증, 암호 해독하기, 파이값 근사하기, 종이 자르기, 복면산 문제, 원의 겹침 판별, 철광석 제련, 3등 찾기, 도서 대출 시스템, 교집합과 합집합, 복리 계산기

## HTML ↔ docx 연동

HTML 강의노트 상단에 인디고 카드 다운로드 버튼을 삽입해서, 학생이 강의노트를 보면서 수행평가 양식(.docx)을 바로 다운로드하도록 연결되어 있다.
