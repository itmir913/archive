## 계기

2024년 10월, 학교생활기록부 작성 교원 연수에서 유의어 점검 방법을 설명하는 장면이 있었다. NEIS 생기부 화면에서 유의어를 **하나씩 직접 쳐서 입력하고 검색**하라는 내용이었다.

유의어 목록이 수십 개인데 하나씩 검색해야 한다. 일괄로 처리해주는 도구는 없었다.

그 자리에서 만들어야겠다는 생각을 했다.

## 첫 구현: Python + PyQt6

2024년 10월에 Python + PyQt6 + PyInstaller로 데스크탑 앱을 만들었다. 생기부 PDF와 Excel을 업로드하면 유의어 목록과 대조해서 발견된 위치를 표시하는 방식이었다.

학교 PC에서 실행하니 문제가 생겼다. Windows AppLocker가 Python 런타임과 Qt DLL을 차단했다. 동작 자체는 됐지만 학교 환경에서 안정적으로 배포할 수 없었다.

Tauri로 재작성해서 해결했다. 기술적 상세는 [학교생활기록부 일괄 점검 프로그램 — 개요와 아키텍처](../school-record-bulk-checker-overview/)에 정리했다.

## teacher-utility-kit의 원형

이 도구가 teacher-utility-kit의 첫 번째 층위다. 학교 교육 활동에서 불필요하게 반복되는 업무를 자동화하겠다는 마인드가 여기서 시작됐다.

유의어를 하나씩 검색하는 일은 생기부 작성의 극히 일부다. 실제 작성 과정 전체를 살펴보면 자동화할 수 있는 것들이 훨씬 많다. 유의어 점검기를 만들면서 그 범위가 눈에 들어왔다.

## School-Record-App으로의 연결

2026년 4월 26일, Claude Pro 구독 첫날 밤새 코딩하면서 School-Record-App 초기 버전을 만들기 시작했다. 유의어 점검기에서 시작된 마인드가 전체 생기부 작성 워크플로를 지원하는 앱으로 확장된 것이다.

점검 도구(WordFinderApp)와 작성 도구(School-Record-App)는 별도 제품으로 유지한다. 두 도구의 사용 시점이 다르고, 대상 사용자의 맥락도 다르기 때문이다. 점검은 작성이 끝난 뒤 전체를 한 번에 훑는 작업이고, 작성은 처음부터 함께 열어두는 환경이다.
