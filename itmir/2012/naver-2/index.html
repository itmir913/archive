부트/리커버리 파티션 찾기

2012년 7월 30일 Android/Kernel 원본 보기 ↗

adb dd 명령어로 부트·리커버리 파티션을 찾는 방법과 unpackbootimg 활용법

안드로이드폰에 CWM 리커버리 이미지를 올린다거나 커스텀 커널을 만들어 부트이미지를 바꾸고자 하는 경우가 있습니다.이 경우 해당 기기를 먼저 루팅해야 하고 그 다음 단계로 부트파티션과 리커버리 파티션을 찾아야 합니다.※ 파티션이라함은 윈도우에서 HDD를 나누어 C:, D:로 사용하는 것처럼 리눅스/안드로이드에서도 내부 저장장치 (MTD혹은 eMMC 장치)의 파티션을 나누어서 사용하는 것을 말합니다부트파티션과 리커버리 파티션에 올릴 수 있는 이미지를 보통 부트 이미지라고 부르는데 이것은 안드로이드 부트로더가 이해할 수 있는 특별한 포맷입니다.↗참고링크: http://android-dls.com/wiki/index.php?title=HOWTO:_Unpack%2C_Edit%2C_and_Re-Pack_Boot_Images이 부트이미지는 unpackbootimg.exe를 이용하여 크게 두가지로 쪼갤 수 있습니다. 1. 리눅스 커널 (압축되어 있으며 보통 5MB 미만) 2. 램디스크라고 불리는 메모리 상에 올라가게 되는 OS구동에 필요한 최소한의 시스템 (압축되어있으며 보통 5MB미만)부트파티션과 리커버리 파티션을 찾는 방식은 이렇게 부트 이미지가 쪼개어질 수 있다는 점을 이용한 것입니다.먼저 파티션 정보를 /proc/partitions 파일로부터 얻고, 3MB 이상 20MB 미만의 크기의 임의의 파티션을 읽어와서 (dd라는 명령을 이용해서 장치를 본을 떠와서 이미지 파일을 얻음) 이미지뜬 파일을 쪼개기를 시도해보는것입니다. 해당 파티션이 부트파티션 혹은 리커버리 파티션인 경우에는 unpackbootimg.exe 명령으로 커널+램디스크 파일로 나뉘어질 수 있습니다.dd 명령 사용법우선 PC와 폰을 연결한 후에 adb shell로 들어갈 수 있어야 합니다.dd if=입력디바이스장치 of=출력파일혹은디바이스예) adb shell dd if=/dev/block/mmcblk0p8 of=/sdcard/my.img이렇게 하면 장치디바이스 /dev/block/mmcblk0p8 (장치디바이스 mmcblk0의 파티션 8번)을 읽어서 /sdcard (저장장치)의 my.img라는 파일 이름으로 다운로드 받는 것입니다. (루팅이 되어 루트 권한이 있어야 실행 가능)(좀 오래된 휴대폰의 경우에는 mmc 장치가 아니라 mtd 장치일 수 있습니다)adb pull 명령으로 /sdcard/my.img를 가져와서 unpackbootimg.exe로 나누기 시도를 하려면adb pull /sdcard/my.img(/sdcard/my.img 파일을 가져옴)unpackbootimg -i my.img(unpackboot.img 명령으로 분해 시도)그러면 반대로 이미지 파일을 장치로 올리려면 어떻게 할까요?방법은 똑같습니다만 입출력 장치를 바꾸면 됩니다. (이경우 출력 장치명을 제대로 입력하지 않으면?? 해당 장치가 내용이 모두 날아갑니다 @@. 백업은 필수, 두번 세번 명령을 확인하세요~)adb pull foo.img /sdcard/foo.imgadb shell$ su (프롬프트 $ 혹은 #는 입력하지 마세요)# dd if=/sdcard/foo.img of=/dev/block/mmcblk0p8램디스크 정보를 통해서 부티파티션/리커버리 파티션 추정unpackbootimg 명령을 통해서 램디스크+커널로 나뉘어지는 이미지는 부팅가능한 부트이미지/리커버리이미지입니다.어떤것이 부트이미지인지 알아내려면 램디스크 파일의 정보를 살펴보아야 합니다.램디스크 파일은 7zip을 이용해서 쉽게 그 내용을 확인하실 수 있습니다.(리눅스의 경우라면 cpio 명령으로 압축을 풀 수 있음)/etc/recovery.fstab 파일과 res/* 파일이 있다면 해당 이미지는 리커버리 이미지이고, 이 이미지를 뜬 파티션이 바로 리커버리 파티션이 됩니다.리커버리 이미지의 램디스크의 특징1. /etc/recovery.fstab파일이 있음2. res/* 파일이 있음3. /sbin/* 실행파일이 좀 더 많음4. init.rc가 단순. 부트이미지에 포함된 램디스크보다 크기가 좀 더 큼부트이미지의 램디스크의 특징일반 모드로 폰을 부팅했을 때에 adb shell로 접속해서 볼 수 있는 최상위 디렉토리 바로 아래에 있는 파일( init.rc 파일 등등)과 구성이 일치합니다. md5sum 명령 체크섬을 확인하면 파일이 완전히 똑같은지 확인하실 수 있습니다.1. init.rc 및 추가의 init..rc가 있고 구성이 복잡함.2. sbin 아래의 실행파일은 adbd / uevent 등등 비교적 적은수의 파일만 있음.FAQQ. 파티션의 개수가 너무 많아 찾기가 어렵습니다. 좀 더 쉬운 방법은 없나요?A. 더 간단한 방법도 물론 있습니다만 약간의 안드로이드 지식이 더 필요합니다.즉, 부트이미지의 헤더는 “ANDROID!”라는 8개의 문자로 시작하게 됩니다. 따라서 부트이미지로 추정되는 영역의 파티션을 8바이트를 읽으면 “ANDROID!” 라는 문자열이 보이게 됩니다.# dd if=/dev/block/mmcblk0p4 bs=1 count=8 | busybox hexdump -Cdd if=/dev/block/mmcblk0p4 bs=1 count=8 |busybox hexdump -C00000000 41 4e 44 52 4f 49 44 21 |ANDROID!|000000088+0 records in8+0 records out8 bytes transferred in 0.004 secs (2000 bytes/sec)위의 예는 프라다의 경우이며 4번 파티션이 부트이미지/리커버리이미지로 추측할 수 있게 되겠지요.이런 방식으로 전체 파티션에 대해서 스크립트를 작성하여 한방에 부트이미지/리커버리 이미지로 추정되는 파티션을 알아낼 수도 있습니다.예를 들어 다음과 같은 식의 스크립트가 되겠지요.#!busyboxfor x in $(ls /dev/block/mmcblk0*); do echo “try $x…” P=$(dd if=$x bs=1 count=8 2>/dev/null) if [ “$P” = “ANDROID!” ]; then echo ” - $x is bootable partition!” fidone첨부파일첨부한 파일에는 안드로이드 SDK에 내장된 adb 명령 실행파일과 본인이 직접 포팅하고 컴파일한 윈도우용 unpackbootimg.exe 실행 파일이 들어있습니다.
